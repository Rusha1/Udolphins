Loaders in React.js

Eric Schwartz
May 15, 2017 · 2 min read

Let’s set the scene: Our app renders a component. This component gets its props from an Ajax call to an API. This often takes a few seconds. During that time, the user sees a blank page. Instead of that blank page, it would be great to give the user some feedback.

Find an appropriate gif.

A Google search led me to this handy little guy:

https://upload.wikimedia.org/wikipedia/commons/b/b1/Loading_icon.gif

Add a conditional statement to your render() function.

We want to make sure that the component has received the props it needs to display on the page before attempting to return the desired content. Assuming the props are represented by an array, let’s do this by checking if the array is empty.

render() {  if (this.props.products.length > 0) {
    return ( 
      <div>
        <MyContent />
      </div>
    )
  }

Render the loading gif.

The above code will break because the render() function must return something, even when the if statement evaluates as false. Let’s fix that by returning our loader gif. Although an else statement would work, we don’t need it because return will break out of the function on its own.

render() {  if (this.props.products.length > 0) {
    return ( 
      <div>
        <MyContent />
      </div>
    )
  }  return (
    <img src="https://upload.wikimedia.org/wikipedia/commons/b/b1/Loading_icon.gif" />
  )}

Summary

Your loading image is now in place! To summarize: Within the render() function, we wrap our content-to-be-returned in a conditional statement to check if the component has received its props from the Ajax request. Following the conditional, we return a stylish loader gif to be displayed while the props have not yet been received. Once the props arrive, React will automatically re-render the component, displaying our content in place of the loader gif.

Edit:

It’s worth mentioning that this method only applies when you can be sure that you’ll be getting some data back from the API. Otherwise, you’ll need to account for empty props if you don’t want your loader to spin forever. Luckily, there’s a quick fix for this scenario:

Assuming you have control of the back end, you can write a conditional that checks for empty data and sends an appropriate response. Perhaps a string that says “No data available.” This way, your props won’t be empty, and the loader will be replaced.

Thomas Andersen suggested a slightly more complex, but foolproof method, which would work with any API and account for errors:

Use a boolean “loading” state variable which is set to “true” just before the request and “false” when the response has arrived. Then do the IF test on state.loading.

Thanks, Thomas!
